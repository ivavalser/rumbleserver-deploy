AWS S3 setup for Rumble Server

Private S3 bucket + dedicated IAM user for file uploads. Choose automatic setup (installer creates everything) or manual setup in the AWS Console.

In the installer AWS step: paste bootstrap keys below, then click Create S3 bucket & IAM user. Bootstrap keys are not saved to .env.

1. Bootstrap IAM user

Bootstrap credentials are used once to create the bucket and scoped app user. Do not use them in the running server.

1.1 Create the user

  1. IAM → Users → Create user
  2. User name: rumbleserver-bootstrap
  3. Programmatic access only — no console login

1.2 Attach permissions

Easiest: attach AWS managed AdministratorAccess (deactivate the user after setup).

Or create a custom policy — copy JSON:

RumbleServerBootstrap 
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Sid": "CreateBuckets",
      "Effect": "Allow",
      "Action": ["s3:CreateBucket"],
      "Resource": "*"
    },
    {
      "Sid": "ConfigureBuckets",
      "Effect": "Allow",
      "Action": [
        "s3:PutBucketPublicAccessBlock",
        "s3:GetBucketLocation",
        "s3:ListBucket"
      ],
      "Resource": "arn:aws:s3:::*"
    },
    {
      "Sid": "ManageAppUser",
      "Effect": "Allow",
      "Action": [
        "iam:CreatePolicy",
        "iam:CreatePolicyVersion",
        "iam:SetDefaultPolicyVersion",
        "iam:ListPolicyVersions",
        "iam:DeletePolicyVersion",
        "iam:CreateUser",
        "iam:AttachUserPolicy",
        "iam:CreateAccessKey",
        "iam:ListAccessKeys",
        "iam:DeleteAccessKey"
      ],
      "Resource": "*"
    },
    {
      "Sid": "Identity",
      "Effect": "Allow",
      "Action": "sts:GetCallerIdentity",
      "Resource": "*"
    }
  ]
}

Public access block action is s3:PutBucketPublicAccessBlock (not s3:PutPublicAccessBlock).

IAM → Policies → Create policy → JSON → name RumbleServerBootstrap → attach to rumbleserver-bootstrap.

1.3 Bootstrap access keys

  1. User rumbleserver-bootstrap → Security credentials
  2. Access keys → Create access key → Application running outside AWS
  3. Copy Access key ID + Secret (secret shown once)
  4. Paste into installer → Bootstrap access key ID / Bootstrap secret access key
After success, delete the bootstrap access key or deactivate rumbleserver-bootstrap.
Re-running auto setup? The installer updates an existing app policy and, if the app IAM user already has 2 access keys, deletes the oldest inactive (or oldest active) key before creating a new one. Bootstrap policy must include iam:ListAccessKeys and iam:DeleteAccessKey (see JSON above).

2. In the installer

  • Enter bootstrap keys + bucket name + region
  • Click Create S3 bucket & IAM user
  • Wait for automatic S3 access verification (may retry while AWS propagates IAM)
  • App keys are filled automatically → click Run automatically to save .env
Create bucket and app IAM user yourself, then paste app credentials into the installer (not bootstrap keys).

1. Choose a region

Pick a region close to your VPS (e.g. eu-north-1). Use the same value in AWS_S3_REGION_NAME.

Open S3 Console →

2. Create a bucket

  1. Create bucket
  2. Name: globally unique, lowercase (e.g. rumble-myserver-prod)
  3. Region: your chosen region
  4. Block all public access — keep enabled ✓

3. IAM policy for the app user

IAM → Policies → Create policy → JSON — copy:

RumbleServerS3-YOUR_BUCKET_NAME 
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Sid": "BucketAccess",
      "Effect": "Allow",
      "Action": [
        "s3:ListBucket",
        "s3:GetBucketLocation"
      ],
      "Resource": "arn:aws:s3:::YOUR_BUCKET_NAME"
    },
    {
      "Sid": "ObjectAccess",
      "Effect": "Allow",
      "Action": [
        "s3:GetObject",
        "s3:PutObject",
        "s3:DeleteObject",
        "s3:AbortMultipartUpload",
        "s3:ListMultipartUploadParts"
      ],
      "Resource": "arn:aws:s3:::YOUR_BUCKET_NAME/*"
    }
  ]
}

ListBucket and GetBucketLocation go on arn:aws:s3:::bucket-name — without /*. Object actions go on arn:aws:s3:::bucket-name/*.

Putting ListBucket only on the /* ARN causes GetBucketLocation ✓ / ListBucket ✗ in the installer.

ListBucket fails but GetBucketLocation works? Open IAM → Policies → RumbleServerS3-… and verify JSON matches the template above.

4. IAM user for the app

  1. IAM → Users → Create user
  2. Name: rumbleserver-s3 (or rumbleserver-s3-YOUR_BUCKET)
  3. Attach the policy from step 3
  4. Security credentials → Create access key → Application running outside AWS
  5. Paste into installer as App access key ID / App secret access key

5. In the installer

  • Region + bucket name
  • App access key ID + secret
  • Check S3 access — verifies GetBucketLocation, ListBucket, PutObject, GetObject, DeleteObject
  • Run automatically — writes .env

Values written to .env

  • AWS_ACCESS_KEY_ID — app user access key
  • AWS_SECRET_ACCESS_KEY — app user secret
  • AWS_STORAGE_BUCKET_NAME — bucket name
  • AWS_S3_REGION_NAME — e.g. eu-north-1
  • AWS_S3_ENDPOINT_URL — https://s3.<region>.amazonaws.com
