AWS S3 setup for Rumble Server

Rumble stores uploaded files (messages, avatars) in S3. You need a private bucket and an IAM user with programmatic access.

Automatic setup: in the installer, use Create S3 bucket & IAM user with bootstrap AWS credentials (IAM user that can create S3 buckets and IAM policies).
Manual setup: follow the steps below, then paste keys into the installer.

1. Choose a region

Pick a region close to your VPS (e.g. eu-north-1, eu-west-1). Use the same region in the installer and in AWS_S3_REGION_NAME.

Open S3 Console →

2. Create a bucket

  1. Click Create bucket
  2. Bucket name: globally unique, lowercase (e.g. rumble-myserver-prod)
  3. Region: your chosen region
  4. Block all public access: keep enabled ✓
  5. Leave other defaults → Create bucket

3. Create IAM policy

IAM → Policies → Create policy → JSON:

{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Sid": "ListBucket",
      "Effect": "Allow",
      "Action": ["s3:ListBucket"],
      "Resource": "arn:aws:s3:::YOUR_BUCKET_NAME"
    },
    {
      "Sid": "ObjectAccess",
      "Effect": "Allow",
      "Action": [
        "s3:GetObject",
        "s3:PutObject",
        "s3:DeleteObject",
        "s3:AbortMultipartUpload",
        "s3:ListMultipartUploadParts"
      ],
      "Resource": "arn:aws:s3:::YOUR_BUCKET_NAME/*"
    }
  ]
}

Name: RumbleServerS3-YOUR_BUCKET_NAME → Create policy.

4. Create IAM user

  1. IAM → Users → Create user
  2. Name: rumbleserver-s3
  3. Attach the policy from step 3
  4. After creation: Security credentials → Create access key → Application running outside AWS
  5. Save Access key ID and Secret access key (shown once)

5. Bootstrap credentials (for automatic setup only)

To use Create S3 bucket & IAM user in the installer, provide a separate IAM user (or admin) with permissions to:

  • s3:CreateBucket, s3:PutPublicAccessBlock
  • iam:CreatePolicy, iam:CreateUser, iam:AttachUserPolicy, iam:CreateAccessKey

Do not use bootstrap keys in production .env — the installer creates a scoped app user.

6. Fill in the installer

  • AWS_ACCESS_KEY_ID — app user access key
  • AWS_SECRET_ACCESS_KEY — app user secret
  • AWS_STORAGE_BUCKET_NAME — bucket name
  • AWS_S3_REGION_NAME — e.g. eu-north-1
  • AWS_S3_ENDPOINT_URL — https://s3.<region>.amazonaws.com

7. Verify

aws s3 ls s3://YOUR_BUCKET_NAME --region YOUR_REGION

After deploy, upload a file via the app and confirm it appears in the bucket.
