AWS S3 setup for Rumble Server

Rumble stores uploaded files (messages, avatars) in a private S3 bucket. The app uses a dedicated IAM user with programmatic access — not your root AWS account.

Two ways to set this up:
A. Automatic — in the installer AWS step, use bootstrap credentials (section 1 below) and click Create S3 bucket & IAM user.
B. Manual — create bucket + app IAM user yourself (sections 2–4), paste app keys into the installer.

1. Bootstrap IAM user (for automatic setup only)

Bootstrap credentials are used once during installation to create the bucket and the scoped app user. They are not stored in .env and must not be used by the running server.

1.1 Create the bootstrap user

  1. Open IAM → Users → Create user
  2. User name: rumbleserver-bootstrap
  3. Do not enable console access — programmatic access only

1.2 Attach permissions

Easiest for a one-time install: attach AdministratorAccess (remove or deactivate the user after setup).

Minimum custom policy for automatic provisioning:

{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Sid": "CreateBuckets",
      "Effect": "Allow",
      "Action": ["s3:CreateBucket"],
      "Resource": "*"
    },
    {
      "Sid": "ConfigureBuckets",
      "Effect": "Allow",
      "Action": [
        "s3:PutBucketPublicAccessBlock",
        "s3:GetBucketLocation",
        "s3:ListBucket"
      ],
      "Resource": "arn:aws:s3:::*"
    },
    {
      "Sid": "ManageAppUser",
      "Effect": "Allow",
      "Action": [
        "iam:CreatePolicy",
        "iam:CreatePolicyVersion",
        "iam:SetDefaultPolicyVersion",
        "iam:ListPolicyVersions",
        "iam:DeletePolicyVersion",
        "iam:CreateUser",
        "iam:AttachUserPolicy",
        "iam:CreateAccessKey"
      ],
      "Resource": "*"
    },
    {
      "Sid": "Identity",
      "Effect": "Allow",
      "Action": "sts:GetCallerIdentity",
      "Resource": "*"
    }
  ]
}

IAM action for public access block is s3:PutBucketPublicAccessBlock (not s3:PutPublicAccessBlock). There is no s3:HeadBucket — bucket checks use s3:ListBucket / s3:GetBucketLocation on the bucket ARN.

IAM → Policies → Create policy → JSON → name RumbleServerBootstrap → attach to rumbleserver-bootstrap.

1.3 Create bootstrap access keys

  1. Open user rumbleserver-bootstrap → tab Security credentials
  2. Access keys → Create access key
  3. Use case: Application running outside AWS
  4. Copy Access key ID and Secret access key (secret is shown once)
  5. Paste them into the installer AWS step → Bootstrap access key ID / Bootstrap secret access key
After automatic setup succeeds, delete the bootstrap access key or deactivate the bootstrap user if you no longer need it.

2. Choose a region

Pick a region close to your VPS (e.g. eu-north-1, eu-west-1). Use the same region in the installer and in AWS_S3_REGION_NAME.

Open S3 Console →

3. Create a bucket (manual path)

  1. Click Create bucket
  2. Bucket name: globally unique, lowercase (e.g. rumble-myserver-prod)
  3. Region: your chosen region
  4. Block all public access: keep enabled ✓
  5. Leave other defaults → Create bucket

4. Create IAM policy for the app user

IAM → Policies → Create policy → JSON:

{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Sid": "BucketAccess",
      "Effect": "Allow",
      "Action": [
        "s3:ListBucket",
        "s3:GetBucketLocation"
      ],
      "Resource": "arn:aws:s3:::YOUR_BUCKET_NAME"
    },
    {
      "Sid": "ObjectAccess",
      "Effect": "Allow",
      "Action": [
        "s3:GetObject",
        "s3:PutObject",
        "s3:DeleteObject",
        "s3:AbortMultipartUpload",
        "s3:ListMultipartUploadParts"
      ],
      "Resource": "arn:aws:s3:::YOUR_BUCKET_NAME/*"
    }
  ]
}

Bucket-level actions (ListBucket, GetBucketLocation) go on arn:aws:s3:::bucket-name — without /*. Object actions go on arn:aws:s3:::bucket-name/*. Putting ListBucket only on the /* ARN is a common mistake and causes exactly GetBucketLocation ✓ / ListBucket ✗ in the installer.

For several servers/buckets, add more ARNs to each Resource array (as in a shared app user policy).

Name: RumbleServerS3-YOUR_BUCKET_NAME → Create policy.

ListBucket fails but GetBucketLocation works?
Open IAM → Policies → RumbleServerS3-… → verify JSON matches the template above (especially s3:ListBucket on the bucket ARN). If you used automatic setup before, run Create S3 bucket & IAM user again — the installer now updates an existing policy instead of silently reusing an old one.

5. Create IAM user for the app

  1. IAM → Users → Create user
  2. Name: rumbleserver-s3 (or rumbleserver-s3-YOUR_BUCKET)
  3. Attach the policy from step 4
  4. After creation: Security credentials → Create access key → Application running outside AWS
  5. Save Access key ID and Secret access key → paste into installer as App access key ID / App secret access key

6. Fill in the installer AWS step

  • Region — e.g. eu-north-1
  • Bucket name
  • App access key ID / App secret access key — from step 5 (or filled automatically after provisioning)

Click Check S3 access — the installer verifies GetBucketLocation, ListBucket, PutObject, GetObject, and DeleteObject on your bucket.

7. Values written to .env

  • AWS_ACCESS_KEY_ID — app user access key
  • AWS_SECRET_ACCESS_KEY — app user secret
  • AWS_STORAGE_BUCKET_NAME — bucket name
  • AWS_S3_REGION_NAME — e.g. eu-north-1
  • AWS_S3_ENDPOINT_URL — https://s3.<region>.amazonaws.com
